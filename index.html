Reduce Database Calls

Your website have 10,000 visitors who are online, and your dynamic page have to send 10,000 same queries to database on every page load. With phpFastCache, your page only send 1 query to DB, and use the cache to serve 9,999 other visitors. 


<?php
    // In your config file
    include("php_fast_cache.php");
    // This is Optional Config only. You can skip these lines.
    // phpFastCache support "apc", "memcache", "memcached", "wincache" ,"files", "pdo" and "mpdo"
    // You don't need to change your code when you change your caching system. Or simple keep it auto
    phpFastCache::$storage = "auto";
    // End Optionals

    // In your Class, Functions, PHP Pages
    // try to get from Cache first.
    $products = phpFastCache::get("products_page");

    if($products == null) {
        $products = YOUR DB QUERIES || GET_PRODUCTS_FUNCTION;
        // set products in to cache in 600 seconds = 5 minutes
        phpFastCache::set("products_page",$products,600);
    }

    foreach($products as $product) {
        // Output Your Contents HERE
    }
?>



Improve cURL & API Calls

You are using Bing API, Google API or cURL and your API transactions are limited? phpFastCache can save your Server CPU & API Transactions. You can use phpFastCache::set() to do it yourself too. 


<?php
    // try to get from Cache first.
    $results = phpFastCache::get("keyword,page");

    if($results == null) {
        $results = phpFastCache::set("keyword,page",
                    cURL->get("http://www.youtube.com/api/json/url/keyword/page"),
                    600);
    }

    foreach($results as $video) {
        // Output Your Contents HERE
    }
?>



Rendering Widgets & Page

phpFastCache help you speed up your website thounsand times by caching your page and widget rendering. 


<?php
    // try to get from Cache first.

Enable Multi Caching Method

phpFastCache support multiple caching methods: Files, PDO, APC, WinCache, MemCache and MemCached. This Example will set up multiple caching files, maximum 50MB per file, and you can SET/Get from APC, MemCache .. etc. It's good for caching cURL Data and API call results. You can mix multiple methods with phpFastCache. For example, mix files caches with APC, or PDO and memcached.


<?php
    // in your config files
    include("php_fast_cache.php");
    phpFastCache::$storage = "auto";
    phpFastCache::$autosize = 50;
    phpFastCache::$path = "/PATH/TO/SECURITY/PLACE/";
    phpFastCache::$securityKey = "yourkeyhere";

    // in your PHP Class, Functions, Pages
    // try to get from Cache first.
    $html1 = phpFastCache::get("keyword1");
    if($html1 == null) {
        $html1 = Render Your Page || Widget || "Hello World";
        phpFastCache::set("keyword1",$html1);
    }

    $html2 = phpFastCache::get(array("files" => "keyword2"));
    if($html2 == null) {
        $html2 = Render Your Page || Widget || "Hello World";
        phpFastCache::set(array("files" => "keyword2"),$html2);
    }

    // Mixing Value & Cache
    $html3 = phpFastCache::get(array("memcached" => "keyword3"));
    if($html3 == null) {
        $html3 = Render Your Page || Widget || "Hello World";
        phpFastCache::set(array("memcached" => "keyword3"),$html3);
    }


    $html3 = phpFastCache::getMulti(array(array("memcached" => "keyword3"),
                                          array("apc" => "keyword"),
                                          "keyword4",
                                    ));
    if($html3['keyword4'] == null) {
        $html3 = Render Your Page || Widget || "Hello World";
        phpFastCache::set("keyword4"),$html3);
    }

    if($html3['apc_keyword'] == null) {
        phpFastCache::setMulti(array(array("memcached" => "keyword3", $html3, 600),
                                     array("apc"    => "keyword", "HELLO WORLD")),
                                3600*24,false);

    }

?>
