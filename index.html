  • X. API Methods

    +

         Description
    $data = phpFastCache::get("keyword")     Return an object from cache or NULL
    phpFastCache::set("keyword",$object,600)
    OR
    phpFastCache::set("keyword",$object,600,true)    		     Set $object to Cache, store in 600 seconds, and TRUE = skip if it is existing in caching system. Return true or false.

  • X. More Addition Methods

    +

    phpFastCache::$storage = "auto"     Enable Multiple Files Cache
    phpFastCache::$autosize = 40     Maximum 40MB per Cache File
    phpFastCache::$path = STRING     Default is blank(automatic detect PATH). Set path to caching.0777 file. If $path = "memory", caching will write in to memory limit by PHP. Memory will be clean up after process web server finish the script.
    phpFastCache::resetAll()     Reset everything in Cache
    phpFastCache::showStats(true|false)     Return an information of caching file as Array.
    phpFastCache::increment("keyword",1)     Increase Value of Object by 1
    phpFastCache::delete("keyword")     Delete an Item by keyword in caching
    phpFastCache::decrement("keyword",1)     Decrease Value of Object by 1
    phpFastCache::getBycURL("keyword",$url,$time)     Get content $URL by cURL. Supported:
    + $time in second
    + $skip if keyword existing
    + $options = array(
         "ref" => "URL REF",
         "user" => "username",
         "pass" => "passwd",
         "hash" => "WHM Access",
         "ssl" => false|true
         "user_agent" => "STRING")
    phpFastCache::postBycURL("keyword",$url,$data)     Post content $URL by cURL. Supported:
    + $data = array("a"=>"val")
    + $time in second
    + $skip if keyword existing
    + $options = array()

Reduce Database Calls

Your website have 10,000 visitors who are online, and your dynamic page have to send 10,000 same queries to database on every page load. With phpFastCache, your page only send 1 query to DB, and use the cache to serve 9,999 other visitors. 


<?php
    // In your config file
    include("php_fast_cache.php");

    // In your Class, Functions, PHP Pages
    // try to get from Cache first.
    $products = phpFastCache::get("products_page");

    if($products == null) {
        $products = YOUR DB QUERIES || GET_PRODUCTS_FUNCTION;
        // set products in to cache in 600 seconds = 5 minutes
        phpFastCache::set("products_page",$products,600);
    }

    foreach($products as $product) {
        // Output Your Contents HERE
    }
?>



Improve cURL & API Calls

You are using Bing API, Google API or cURL and your API transactions are limited? phpFastCache supported cURL and automatic cache to system to save your Server CPU, Memory and API Transactions. You can use phpFastCache::set() to do it yourself too. 


<?php
    // try to get from Cache first.
    $results = phpFastCache::get("keyword,page");

    if($results == null) {
        $results = phpFastCache::getBycURL("keyword,page", "http://www.youtube.com/api/json/url/keyword/page", 600);
    }

    foreach($results as $video) {
        // Output Your Contents HERE
    }
?>



Rendering Widgets & Page

<?php
    // try to get from Cache first.

    if($html == null) {
        $html = Render Your Page || Widget || "Hello World";
        phpFastCache::set("keyword,page",$html);
    }

    echo or return your $html;
?>



Enable Multi Caching Files

If you want to cache so many data, I suggest you enable Multi Caching Files. This Example will set up multiple caching files, and maximum 50MB per file. It's good for caching cURL Data and API call results.


<?php
    // in your config files
    include("php_fast_cache.php");
    phpFastCache::$storage = "auto";
    phpFastCache::$autosize = 50;
    phpFastCache::$path = "";

    // in your PHP Class, Functions, Pages
    // try to get from Cache first.
    $html = phpFastCache::get("keyword,page");

    if($html == null) {
        $html = Render Your Page || Widget || "Hello World";
        phpFastCache::set("keyword,page",$html);
    }

    echo or return your $html;
?>
